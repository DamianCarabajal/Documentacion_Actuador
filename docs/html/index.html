Sistema IoT de Control de Temperatura – Nodo Sensor + Nodo Actuador

¿Qué es este sistema?

Este proyecto implementa un sistema IoT sencillo para el control de temperatura en una sala, utilizando dos nodos físicos y un broker MQTT:

  • Un Nodo de Sensado (ESP8266) que mide temperatura y humedad.
  • Un Nodo Actuador (ESP32-C6) que mueve un servomotor según una consigna.
  • Un broker MQTT (por ejemplo, una Raspberry Pi) que concentra toda la información y permite la integración con herramientas como Node-RED.

El objetivo es disponer de un ejemplo completo y didáctico que combine:

  • Adquisición de datos.
  • Control PI discreto.
  • Comunicación IoT (WiFi + MQTT).
  • Ejecución en tiempo real con FreeRTOS (en el nodo actuador).

Arquitectura general

A alto nivel, la arquitectura puede describirse así:

  • Nodo_de_Sensado (ESP8266):
    • Mide temperatura y humedad mediante un sensor DHT11.
    • Se conecta a una red WiFi de laboratorio.
    • Resuelve dinámicamente la IP del broker (por nombre de host).
    • Publica periódicamente las mediciones vía MQTT.
    • Utiliza LWT (Last Will and Testament) para informar si está Online o Offline.
  • Nodo_Actuador (ESP32-C6):
    • Implementa un controlador PI discreto diseñado en MATLAB y discretizado por método de Tustin.
    • Recibe por MQTT:
      • La consigna de temperatura (SP).
      • La temperatura medida (PV).
      • Los parámetros de sintonía Kp y Ti.
    • Calcula una salida lógica (0–100 %) y la convierte a una posición en grados de un servomotor (0–180°).
    • Publica su estado (posición del servo y mando lógico) a través de MQTT.
    • Ejecuta el lazo de control y las comunicaciones en tareas separadas de FreeRTOS.
  • Broker_MQTT:
    • Centraliza todos los tópicos.
    • Permite visualizar y manipular los datos desde Node-RED, dashboards, u otras aplicaciones.

Tópicos MQTT utilizados

Tópicos del Nodo de Sensado (Nodo de Sensado)

Tópico base:

  • universidad/piso3/sala_de_profes

Tópicos derivados:

  • universidad/piso3/sala_de_profes/temperatura
    • Publica la temperatura medida (en °C, con 1 decimal).
  • universidad/piso3/sala_de_profes/humedad
    • Publica la humedad relativa (en %, sin decimales).
  • universidad/piso3/sala_de_profes/status
    • Publica el estado del nodo: Online, Offline (LWT) o ErrorSensor.

Tópicos del Nodo Actuador (NODE_ACTUATOR)

Tópicos de suscripción (entradas):

  • emu/t_sp
    • Consigna de temperatura (SP).
  • emu/t_act
    • Temperatura de proceso (PV).
  • emu/kp_set
    • Valor de Kp para el controlador PI.
  • emu/ti_set
    • Valor de Ti (tiempo integral) para el controlador PI.

Tópicos de publicación (salidas):

  • emu/servo_pos
    • Posición física del servomotor (grados).
  • emu/u_out
    • Mando lógico del PI (0–100 %).

Todos los mensajes se envían como texto ASCII, lo que facilita la depuración y el trazado de datos desde Node-RED u otras herramientas.

Flujo típico de funcionamiento

  1. El Nodo de Sensado se conecta a WiFi, resuelve la IP del broker MQTT y comienza a publicar temperatura y humedad cada cierto intervalo.
  2. El NODE_ACTUATOR se conecta a WiFi y al mismo broker MQTT.
  3. Desde un PC o una interfaz (por ejemplo Node-RED) se envía:
    • La consigna de temperatura (SP).
    • Opcionalmente, la sintonía del PI (Kp y Ti).
  4. El NODE_ACTUATOR:
    • Recibe la SP y la PV.
    • Ejecuta el controlador PI discreto.
    • Ajusta la posición del servomotor en consecuencia.
    • Va publicando su estado (posición y mando).
  5. El sistema completo puede visualizarse y supervisarse desde:
    • Dashboards.
    • Gráficos de tendencia.
    • Scripts de MATLAB analizando el log del actuador.
